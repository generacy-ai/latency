# Feature Specification: Update workflow verification to use discovery-based validation

**Branch**: `048-problem-verification-phase` | **Date**: 2026-03-18 | **Status**: Draft

## Summary

Replace hardcoded `pnpm run test` and `pnpm run lint` verification steps in `.generacy/speckit-feature.yaml` and `.generacy/speckit-bugfix.yaml` with the `build.validate` tool, which auto-detects the package manager and discovers all available validation scripts.

## Problem

The verification phase in both workflow files hardcodes `pnpm` and only runs two checks (`test` and `lint`):

```yaml
- name: run-tests
  uses: verification.check
  with:
    command: pnpm run test

- name: run-lint
  uses: verification.check
  with:
    command: pnpm run lint
```

This has two issues:
1. **Assumes pnpm** — breaks for projects using npm, yarn, or other package managers.
2. **Misses validation scripts** — format checking (`format:check`), type checking (`typecheck`), and any other configured validation scripts are not run.

## Proposed Solution

**Depends on:** generacy-ai/agency#323 (`build.validate` tool)

Replace the two hardcoded `verification.check` steps with a single `build.validate` step. The `build.validate` tool:
- Auto-detects the project's package manager
- Discovers available validation scripts (lint, format:check, typecheck, test, etc.)
- Runs all discovered validation commands

## Files to modify

- `.generacy/speckit-feature.yaml`
- `.generacy/speckit-bugfix.yaml`

## User Stories

### US1: Package-manager-agnostic verification

**As a** developer using speckit workflows in a project that may use npm, yarn, or pnpm,
**I want** the verification phase to auto-detect my package manager,
**So that** I don't have to manually configure or override verification steps per project.

**Acceptance Criteria**:
- [ ] Verification phase does not hardcode any specific package manager
- [ ] `build.validate` tool is used instead of manual `verification.check` steps

### US2: Comprehensive validation coverage

**As a** developer with multiple validation scripts configured (lint, format:check, typecheck, test),
**I want** the verification phase to discover and run all available validation scripts,
**So that** no configured checks are skipped during workflow verification.

**Acceptance Criteria**:
- [ ] All available validation scripts (not just test and lint) are discovered and run
- [ ] The verification phase runs `build.validate` which handles discovery automatically
