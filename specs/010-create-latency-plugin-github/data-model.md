## ID Mapping

| Context | Type | Example |
|---------|------|---------|
| IssueTracker interface | `string` | `"42"` |
| GitHub API | `number` | `42` |
| GitHubIssue.id | `string` | `"42"` (stringified number) |
| GitHubIssue.number | `number` | `42` (native GitHub number) |

The plugin parses string IDs to numbers for API calls and stringifies numbers for the interface.

## Validation Rules

- **Issue ID**: Must parse to a positive integer
- **GitHubConfig.token**: Non-empty string
- **GitHubConfig.owner**: Non-empty string
- **GitHubConfig.repo**: Non-empty string
- **IssueSpec.title**: Non-empty (validated by AbstractIssueTrackerPlugin base class)
- **Comment body**: Non-empty (validated by base class)
- **Labels**: Array of non-empty strings
